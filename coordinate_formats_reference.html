F1 — x/y/w/h
F2 — bbox
F3/4 — rectangle
F5 — coordinates
F6 — min/max
F7 — polygon
F8 — left/right
F9 — edges
Normalise
Format 1 Flat x/y — most common. Width aliases: w or width. Height: h or height.
// Input (fractions 0-1 OR absolute pixels/pts)
{ x: 0.10, y: 0.12, width: 0.25, height: 0.03 }
{ x: 72,   y: 86,   w: 180,         h: 21          }

// extractRawCoords → always returns [x, y, w, h]
return [item.x, item.y,
        (item.width ?? item.w),
        (item.height ?? item.h)];
Format 2 bbox array — two sub-forms. Auto-detects which by checking if c>a AND d>b (corner points) vs treating as x,y,w,h.
// Sub-form A: [x, y, w, h]
{ bbox: [72, 86, 180, 21] }

// Sub-form B: [x1, y1, x2, y2] corner points
{ bbox: [72, 86, 252, 107] }

// Detection logic:
const [a, b, cc, d] = item.bbox;
if (cc > a && d > b && (a>1 || b>1 || cc>1 || d>1)) {
  // corner-point form → convert to x,y,w,h
  return [a, b, cc-a, d-b];
}
return [a, b, cc, d]; // already x,y,w,h
⚠ If pageWidth is provided on the item, cc is compared to pageWidth×0.95 to confirm corner vs width.
Format 3/4 rectangle array — same two sub-forms as bbox, same detection logic.
// Sub-form A: [x, y, w, h]
{ rectangle: [0.10, 0.12, 0.25, 0.03] }

// Sub-form B: [x1, y1, x2, y2] corner points
{ rectangle: [72, 86, 252, 107] }

// Same detection as bbox:
const [a, b, c, d] = item.rectangle;
if (c > a && d > b && (a>1||b>1||c>1||d>1))
  return [a, b, c-a, d-b]; // corner → x,y,w,h
return [a, b, c, d];
Format 5 coordinates array in ymin, xmin, ymax, xmax order — note axis swap (y first).
// Input: [ymin, xmin, ymax, xmax]
{ coordinates: [86, 72, 107, 252] }
//              ymin xmin ymax  xmax

// extractRawCoords — swap axes, convert to x,y,w,h:
const [ymin, xmin, ymax, xmax] = item.coordinates;
return [xmin, ymin, xmax-xmin, ymax-ymin];
//      x     y     w           h
⚠ Axis order is Y-first (common in ML/AI output formats). The viewer swaps back to X-first internally.
Format 6 Flat min/max — four named fields covering the bounding box corners.
// Input
{ xmin: 72, ymin: 86, xmax: 252, ymax: 107 }

// extractRawCoords:
const x = item.xmin, y = item.ymin;
return [x, y, item.xmax - x, item.ymax - y];
Format 7 bbox_relative polygon — array of [x,y] point pairs (3–4+ points). Viewer computes the axis-aligned bounding box (AABB) — the smallest rectangle that encloses all points.
// Input: 4-corner polygon [[x,y],[x,y],[x,y],[x,y]]
{ bbox_relative: [
    [0.10, 0.12],   // top-left
    [0.35, 0.12],   // top-right
    [0.35, 0.15],   // bottom-right
    [0.10, 0.15]    // bottom-left
  ]
}

// extractRawCoords — compute AABB from all points:
const pts = item.bbox_relative;
const xs  = pts.map(p => p[0]);
const ys  = pts.map(p => p[1]);
return [
  Math.min(...xs), Math.min(...ys),           // x, y (top-left)
  Math.max(...xs) - Math.min(...xs),          // width
  Math.max(...ys) - Math.min(...ys),          // height
];
Works for any polygon (triangle, quad, rotated rect). Non-axis-aligned shapes lose rotation — only the enclosing rectangle is highlighted.
Format 8 left/right/width/height — common in OCR engines. left=x origin, width preferred over deriving from right. top used as y if present.
// With top (full position)
{ left: 72, top: 86, width: 180, height: 21 }

// Without top (y defaults to 0)
{ left: 72, right: 252, height: 21 }

// extractRawCoords:
const l = item.left;
const y = typeof item.top === 'number' ? item.top : 0;
const w = typeof item.width === 'number'
  ? item.width
  : item.right - l;       // derive from right if no width
return [l, y, w, item.height];
Format 9 Four edge values — all four sides of the bounding box explicitly. No width/height field present.
// Input: all four edges
{ left: 72, top: 86, right: 252, bottom: 107 }

// extractRawCoords:
const l = item.left,  t = item.top;
const r = item.right, b = item.bottom;
return [l, t, r - l, b - t];
//      x  y  width   height
Differs from Format 8 by the absence of width/height — the viewer uses this to distinguish the two.
After extraction, all formats produce [x, y, w, h]. These are then normalised to 0–1 fractions using the page dimensions. The viewer draws at: canvas_x = x × pagePixelWidth, canvas_y = y × pagePixelHeight.
// All values 0-1 → already fractional, pass through
[0.10, 0.12, 0.25, 0.03] → stored as-is

// Absolute values → divide by page dimensions
// Page dims come from the loaded document (not hardcoded):
//   PDF:   pdfjs page.view in pts (e.g. 612×792 for A4)
//   TIFF:  decoded pixel size (e.g. 1700×2200 at 300dpi)
//   DOCX:  DocAdapter canvas 794×1123px
dims = adapter.getPageDimensions(page); // {width, height}
scale = detectBestUnitScale(rawMax, dims); // pts/px/inch/cm
x_norm = rawX * scale / dims.width;
y_norm = rawY * scale / dims.height;
w_norm = rawW * scale / dims.width;
h_norm = rawH * scale / dims.height;

// Rendering on canvas (PDFViewer, ImageViewer)
ctx.fillRect(
  x_norm * canvasW,   // pixel x on screen
  y_norm * canvasH,   // pixel y on screen
  w_norm * canvasW,   // pixel width
  h_norm * canvasH,   // pixel height
);
